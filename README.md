## 🎯 Business Problem

Businesses often struggle with fragmented data across systems, making it difficult to generate reliable insights.

This project solves that problem by:

- Building a structured **data warehouse**
- Transforming raw data into **analysis-ready datasets**
- Generating **actionable insights** for decision-making

## 📊 Project Overview

This project focuses on designing and implementing a **data warehouse** using SQL.

Key components:

- Data cleaning and transformation  
- Creation of **fact and dimension tables**  
- Use of **views and stored procedures**  
- Preparing structured datasets to support business reporting and KPI generation  

---

- SQL Server (Database)

- ETL Concepts (Extract, Transform, Load)

## 💡 Key Insights

- Designed a star schema with fact and dimension tables for optimized querying  
- Transformed raw data into clean, analysis-ready datasets  
- Built reusable SQL views to simplify downstream analytics  
- Structured data to support efficient aggregation and reporting
